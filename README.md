# Εργασία Προγραμματισμού σε Python  (Οδηγίες για τους φοιτητές)
## Υλοποίηση σεναρίου χρήσης (use case) συστήματος

> Κάθε ομάδα έχει αναλάβει **ένα συγκεκριμένο σενάριο** (θέμα) συστήματος
> από αυτά που δόθηκαν στην προηγούμενη ενότητα ανάλυσης (use cases, class diagrams, ER).
> Στην εργασία αυτή θα το υλοποιήσει σε Python.

---

## 1. Στόχος

Σκοπός της εργασίας είναι:

- να συνδέσετε την **ανάλυση απαιτήσεων & σχεδίαση** (use cases, class diagram, ER)  
  με την **υλοποίηση** (Python κλάσεις, συναρτήσεις, δομές ελέγχου),
- να υλοποιήσετε ένα **πλήρες σενάριο χρήσης** από την αρχή μέχρι το τέλος  
  για το **σενάριο που έχει ανατεθεί στην ομάδα σας**.

Η εφαρμογή θα είναι **προγραμματάκι κονσόλας (CLI)** ή απλή εφαρμογή με μενού.

---

## 2. Τι ακριβώς πρέπει να υλοποιήσετε

1. Χρησιμοποιείτε το **ανατεθειμένο σενάριο** (Assigned Scenario) της ομάδας σας.  
   Παραδείγματα τύπων σεναρίων (μόνο ενδεικτικά):
   - ηλεκτρονική έκδοση εγγράφου,
   - διαχείριση συμβολαίων / αιτημάτων,
   - σύστημα κρατήσεων,
   - σύστημα παρακολούθησης αιτήσεων, κ.ά.

2. Επιλέγετε **ένα βασικό use case** (ή ένα καλά οριοθετημένο υποσύνολο)     και το υλοποιείτε από την αρχή μέχρι το τέλος (happy path, και όπου προλαβαίνετε βασικούς ελέγχους λαθών).

3. Η εφαρμογή σας πρέπει να:
   - δέχεται δεδομένα από τον χρήστη (input),
   - δημιουργεί και χειρίζεται **αντικείμενα** βάσει των κλάσεων του διαγράμματός σας,
   - εκτελεί τη λογική του σεναρίου με `if`, `for`, `while`, συναρτήσεις κτλ.,
   - εμφανίζει καθαρό αποτέλεσμα (π.χ. επιβεβαίωση, λίστα εγγραφών, αναφορά).

---

## 3. Παραδοτέα

### 3.1. Σχεδίαση (PDF & εικόνα)

- Ενημερωμένο **class diagram** ΜΟΝΟ με τις κλάσεις που χρησιμοποιείτε στην υλοποίηση:
  - π.χ. `User`, `Request`, `Document`, `SystemService` κ.λπ.
  - ένα **use case diagram**,  
  - και 1–2 περιγραφές σεναρίων (βήματα happy path σε κείμενο).

### 3.2. Κώδικας Python

- Τουλάχιστον **2–3 κλάσεις** που αντιστοιχούν σε έννοιες του συστήματος.
- Κατάλληλα **attributes** και **methods** σύμφωνα με το class diagram.
- Χρήση **συναρτήσεων** και δομών ελέγχου (`if`, `for`, `while`) για να υλοποιηθεί το σενάριο.
- Ο κώδικας να είναι οργανωμένος σε αρχεία, π.χ.:

  ```text
  models.py     # κλάσεις τομέα (domain classes)
  services.py   # λογική συστήματος / βοηθητικές συναρτήσεις
  main.py       # μενού / διάλογος με τον χρήστη
    ```

Το main.py πρέπει να μπορεί να τρέξει ως:

```bash
python main.py
```

και να οδηγεί τον χρήστη βήμα–βήμα στην εκτέλεση του use case. Δεν απαιτείται βάση δεδομένων. Μπορείτε να χρησιμοποιήσετε:

- λίστες, λεξικά, κλάσεις in-memory,
- προαιρετικά αποθήκευση σε JSON/CSV αρχείο.

### 3.3. **Αρχείο οδηγιών (`README.md`)**

   - Σύντομη περιγραφή του σεναρίου.
   - Πώς τρέχω το πρόγραμμα (εντολές).
   - Παράδειγμα μιας τυπικής εκτέλεσης (π.χ. screenshot ή αντιγραφή από το terminal).

## 4. Προτεινόμενα βήματα υλοποίησης

1. **Επιλογή σεναρίου & επανάληψη UML**
   - Ξαναδιαβάστε το κείμενο του θέματος.
   - Επιλέξτε *ποιο* ακριβώς use case θα υλοποιήσετε (π.χ. «Έκδοση εξουσιοδότησης»).

2. **Ενημέρωση class diagram**
   - Επιλέξτε τις βασικές κλάσεις (2–5).
   - Καθορίστε βασικά attributes και σχέσεις (π.χ. `Citizen` – `Document`).

3. **Μετατροπή κλάσεων UML → κλάσεις Python**
   - Δημιουργήστε αρχείο `models.py`.
   - Γράψτε τις κλάσεις με `__init__` και βασικές μεθόδους.

4. **Σχεδίαση ροής σεναρίου**
   - Γράψτε σε ψευδοκώδικα ή σχόλια την ακολουθία βημάτων:
     - είσοδος, συμπλήρωση στοιχείων, αποθήκευση, εμφάνιση αποτελέσματος, αναζήτηση κτλ.
   - Χωρίστε τη ροή σε συναρτήσεις σε αρχείο `services.py`.

5. **Υλοποίηση `main.py`**
   - Γράψτε μενού ή διαδοχικές ερωτήσεις στον χρήστη.
   - Καλέστε συναρτήσεις / μεθόδους ώστε να υλοποιείται το use case.

6. **Δοκιμή με 2–3 διαφορετικά σενάρια**
   - Π.χ. 2 διαφορετικούς πολίτες, διαφορετικές δηλώσεις κ.λπ.
   - Ελέγξτε τι γίνεται σε λανθασμένα δεδομένα (άδειο πεδίο, λάθος επιλογή μενού).

## 5. Τεχνικές διευκρινίσεις
 
- Γλώσσα: **Python 3.10+**
- Δεν απαιτείται βάση δεδομένων – μπορείτε να χρησιμοποιήσετε:
  - απλές μεταβλητές,
  - λίστες / λεξικά in-memory,
  - **προαιρετικά** αποθήκευση σε JSON/CSV.
- ΔΕΝ υλοποιούμε πραγματική σύνδεση με taxisnet, email, SMS κλπ – αυτά προσομοιώνονται με μηνύματα στην οθόνη.
- Ο κώδικας να είναι χωρισμένος σε **αρχεία / modules**:
  - π.χ. `models.py` για τις κλάσεις,
  - `services.py` για την «λογική συστήματος»,
  - `main.py` για το μενού / είσοδο χρήστη.

Δεν απαιτούνται:

- Γραφικό περιβάλλον (GUI) ή web framework.
- Πραγματικά συστήματα τρίτων (π.χ. taxisnet, email, SMS, API κ.λπ.). Αυτά προσομοιώνονται με απλά `print()` μηνύματα.

Μπορείτε να χρησιμοποιήσετε απλές βιβλιοθήκες της standard library αν σας βοηθούν
(π.χ. datetime, json, csv).

## 6. Βαθμολόγηση (ενδεικτικά)

- 20% – Σωστή χαρτογράφηση UML → κώδικα (κλάσεις και δομές δεδομένων)
- 40% – Ποιότητα υλοποίησης σε Python
  (ορθή χρήση κλάσεων / συναρτήσεων, δομές ελέγχου, καθαρός κώδικας)
- 20% – Ροή σεναρίου από την αρχή μέχρι το τέλος (happy path)
- 10% – Αναγνωσιμότητα / σχόλια / ονόματα μεταβλητών
- 10% – `README` και οδηγίες εκτέλεσης


## 7. Τι **δεν** χρειάζεται (για να μην αγχωθείτε)

- Δεν χρειάζεται γραφικό περιβάλλον (GUI) ή web περιβάλλον.
- Δεν χρειάζεται πραγματική βάση δεδομένων ή APIs.
- Δεν εξετάζονται πολύπλοκα patterns ή προχωρημένες βιβλιοθήκες.


## 8. Παράδοση μέσω GitHub

Κάθε ομάδα θα παραδώσει την εργασία της σε **GitHub repository**.

- 1 repository ανά ομάδα.
- Προτεινόμενη ονομασία: `efp1-2025-assignment-teamXX`
- Μέλη ομάδας να αναφέρονται στο README.

### 8.1 Τι πρέπει να περιέχει το repository

- Τον κώδικα Python (`main.py`, `models.py`, `services.py`, κ.λπ.).
- Τα διαγράμματα (class diagram, sequence diagram) σε εικόνα ή PDF
  μέσα σε φάκελο `diagrams/`.
- Ένα `README.md` με:
  - σύντομη περιγραφή του σεναρίου,
  - οδηγίες εκτέλεσης,
  - παράδειγμα εκτέλεσης (input/output).

### 8.2 Τρόπος παράδοσης

- Ένα μέλος της ομάδας στέλνει στο eClass:
  - το **link** του GitHub repository,
  - και τα ονόματα / ΑΜ όλων των μελών.

### 8.3 Ενδεικτικό GitHub workflow για την ομάδα

1. Ένα μέλος δημιουργεί το repository στο GitHub.
2. Προσθέτει τα υπόλοιπα μέλη ως collaborators.
3. Όλοι κάνουν:

   ```bash
   git clone <URL του repository>
   ```

###  8.4 Τυπικός κύκλος δουλειάς:

```bash
# Ενημέρωση από το remote
git pull

# Δουλεύω σε ένα αρχείο, π.χ. models.py

# Προσθήκη αλλαγών
git add models.py

# Commit με μήνυμα
git commit -m "Προσθήκη κλάσης Citizen"

# Ανεβάζω στο GitHub
git push
```
